## 学習

baseline configではtemplate 2枚、search 2フレームを1 forwardへ渡す。modelは`track_query`というmodule fieldを保持し、各search frame後にbackboneのtemporal tokenを`clone().detach()`して次search frameへ渡す。各search frameのGIoU/L1/focal lossを合計する。

| 項目 | 確認結果 | 根拠 |
| --- | --- | --- |
| sample | template 2、search 2 | `experiments/mambalct/baseline_256.yaml:7-22` |
| context更新 | temporal tokenを次searchへ渡す | `models/mambalct/mambalct.py:33-72` |
| gradient | `track_query`はupdate時にdetach | `mambalct.py:55-59` |
| loss時刻 | 各search frameでGIoU/L1/focalを合算 | `actors/mambalct.py:36-73, 75-136` |
| explicit reset | inspected actor/model pathに`track_query`をsampleごとに明示resetする箇所はない | `mambalct.py:33-72`, `actors/mambalct.py:36-73` |

この実装のMambaはtemplate/search内のtoken処理に使われる。動画をまたぐ長期文脈の主実体は、Mambaの`InferenceParams` cacheではなく、detachしたtemporal queryとmemory frame群である。

## 推論

inferenceは前回prediction `self.state`を中心にsearch cropを切り出す。各frameのpredictionを用いてtemplate cropを作り、`memory_frames`へ追加する。次frameでは、初期frameと時間を分割したmemory frameを選びtemplateとして使う。確認した実装ではtemplate memory更新にscore gateはなく、bbox predictionを用いたframe追加は無条件である。

| 項目 | 確認結果 | 根拠 |
| --- | --- | --- |
| state | `self.state`、`memory_frames`、model field `track_query` | `test/tracker/mambalct.py:49-69`; `models/mambalct/mambalct.py:33-72` |
| prediction入力 | 前回`self.state`でsearch cropを作る | `mambalct.py:75-109` |
| memory update | prediction bboxからtemplate cropを作り無条件append | `mambalct.py:111-127` |
| long-term selection | initial frame + time-segmented memory frameを選択 | `mambalct.py:166-195` |
| low-confidence guard | inspected tracker pathにはstate/memory updateのscore gateなし | `mambalct.py:101-127` |
| matching / accepted detection | SOTのため存在しない | tracker全体 |

## 現行MambaStatefulへの示唆

- Mambaを使うtrackerでも、必ずしもMamba cacheをtrack間で逐次持ち越すわけではない。明示的な短期queryや選択的memoryで長期文脈を作る設計もある。
- `detach()`したcontextは長期情報をforwardへ渡せても、長期全体を通るgradientを要求しない。これはTBPTT候補の一形態として参考になる。
- 一方、MambaLCTのself-prediction由来memory無条件更新は、MOTのstate汚染対策にはそのまま採用できない。観測信頼度を使うMCITrackのguardと対照的である。
