## 学習

MCITrackは、template 5枚とsearch 2フレームをsampleし、Mamba neckのhidden stateを2 search frameにまたがって更新する。actorはsampleごとに`neck_h_state = [None] * N_LAYERS`から開始し、各search frameでencoder → neck → decoderを順に実行する。各frameのbox / score-map lossを合計するため、state carryを含む短い時系列unrollへ全時刻supervisionを与えている。

| 項目 | 確認結果 | 根拠 |
| --- | --- | --- |
| sample | visible frameからtemplate 5、search 2をsampling | `experiments/mcitrack/mcitrack_s224.yaml:7-22`, `sampler.py:12-38, 123-179` |
| state初期化 | sampleごとに`None` | `actors/mcitrack.py:33-43` |
| state carry | search frame loopで`neck_h_state`を次stepへ渡す | `actors/mcitrack.py:39-46` |
| loss時刻 | 各search frameのGIoU/L1/focal lossを合算 | `actors/mcitrack.py:50-95` |
| BPTT | actor内でdetachせず2-step state pathを展開 | `actors/mcitrack.py:33-48` |

## 推論

SOT trackerは前回prediction `self.state`を中心にcurrent search cropを作る。Mamba neckの`h_state`はframe間で更新するが、confidenceがdataset依存閾値`UPH`を下回ると全layerを`None`へresetする。さらに高confidence時だけtemplate memoryを追加し、memory bank上限を超えれば古いtemplateを削除する。

| 項目 | 確認結果 | 根拠 |
| --- | --- | --- |
| track state | layerごとの`h_state`を保持 | `test/tracker/mcitrack.py:29-35, 104-133` |
| prediction入力 | 前回`self.state`でsearch cropを作る | `mcitrack.py:92-110` |
| low-confidence guard | `conf_score < update_h_t`でhidden stateをreset | `mcitrack.py:41-59, 130-133`; config `:82-89` |
| memory guard | high confidence時のみtemplateを追加 | `mcitrack.py:135-164` |
| matching / accepted detection | SOTのため存在しない | tracker全体 |

## 現行MambaStatefulへの示唆

MCITrackは、今回の調査対象で最も近い「学習時にも短い時系列state carryを実際に展開する」例である。直接移植候補は、次の二点である。

- 学習: track/sequence内の複数時刻にstateを渡し、各時刻へlossを置く。
- 推論: low-confidenceに応じてstateをresetまたはstate updateを拒否する。

ただしconfidenceはSOT headのscoreであり、MOTではmatched detection score、prediction-consistency、IoU、欠測長などから別途定義する必要がある。MCITrackはMOT matchingやaccepted detectionの扱いを提供しない。
